# 프로젝트 킥오프 워크플로우

새 프로젝트의 분석-설계 프로세스를 시작합니다.

## 워크플로우 개요

```
┌─────────────────────────────────────────────────────────────┐
│                   프로젝트 킥오프 워크플로우                  │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Step 1: 프로젝트 개요 수집                                  │
│  └─▶ 이 스킬에서 기본 정보 수집                              │
│                                                             │
│  Step 2: 요구사항 분석                                       │
│  └─▶ @requirements-analyst 호출                             │
│       출력: PRD (요구사항 명세서)                            │
│                                                             │
│  Step 3: 실험 설계                                          │
│  └─▶ @experiment-designer 호출                              │
│       출력: 실험 설계서 (메트릭, A/B 테스트 계획)            │
│                                                             │
│  Step 4: 시스템 설계                                        │
│  └─▶ @system-architect 호출                                 │
│       출력: 아키텍처 설계서                                  │
│                                                             │
│  Step 5: 구현                                               │
│  └─▶ @implementer 호출                                      │
│       - 태스크 분해 및 TODO 생성                            │
│       - 설계 기반 코드 구현                                  │
│       - 테스트 작성 및 검증                                  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 시작하기

프로젝트에 대해 간단히 설명해주세요:

1. **프로젝트명**: 이 프로젝트의 이름은?
2. **한 줄 설명**: 이 프로젝트가 해결하려는 문제는?
3. **프로젝트 유형**:
   - [ ] 전통 ML (분류/회귀/추천/검색)
   - [ ] LLM/GenAI (RAG/챗봇/생성)
   - [ ] 백엔드 API
   - [ ] 하이브리드

설명을 받으면 `@requirements-analyst`를 호출하여 상세 분석을 시작합니다.

---

## 사용 예시

```
/kickoff

프로젝트: 고객 리뷰 감성 분석 시스템
- 고객 리뷰 텍스트를 분석해서 긍정/부정/중립을 분류
- 실시간 API로 서비스
- 하루 10만 건 처리 예상
```

이렇게 입력하시면 자동으로 분석-설계 프로세스가 시작됩니다!
